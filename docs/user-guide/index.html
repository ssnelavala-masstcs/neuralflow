User Guide

A visual walkthrough of the NeuralFlow desktop interface — the canvas, node palette, properties panel, run history, memory management, and settings.

Overview

NeuralFlow is a Tauri v2 desktop application. When you launch it, you'll see a full-screen canvas with a dot-grid background, a node palette on the left, and a toolbar across the top. The Python sidecar runs silently in the background on port 7411.

🖥️

Canvas

Drag-and-drop workspace where you build workflows by placing and connecting nodes.

📋

Node Palette

Left sidebar with all 10 node types organized by category: Agent, Tool, and Data nodes.

⚙️

Properties Panel

Right sidebar that slides in when you select a node. Configure model, prompts, and parameters.

The Canvas

The canvas is your primary workspace. It displays all nodes in your current workflow as draggable cards connected by directed edges (arrows).

  • Pan: Click and drag on empty canvas space to pan the viewport.
  • Zoom: Use the mouse scroll wheel or pinch-to-zoom on a trackpad.
  • Select: Click a node to select it. The Properties panel opens on the right.
  • Multi-select: Hold Shift and click to select multiple nodes.
  • Delete: Press Delete or Backspace to remove selected nodes.
  • Connect: Drag from an output handle (right edge) to an input handle (left edge) of another node.

Nodes Panel

The node palette on the left side of the canvas lists all available node types grouped by category. Drag any node onto the canvas to add it to your workflow.

  • Agent Nodes: AgentNode, RouterNode, HumanNode
  • Tool Nodes: ToolNode, McpNode
  • Data Nodes: InputNode, OutputNode, TransformNode, FilterNode, AggregatorNode
Tip: See the Node Reference for a complete description of every node type and its properties.

Properties Panel

When you select a node on the canvas, the Properties panel slides in from the right. It shows all configurable fields for that node type with inline validation and help text.

  • Label: The display name shown on the canvas node card.
  • Model: (AgentNode only) Select from available providers and models.
  • System Prompt: The system message for LLM inference calls.
  • Tools: Multi-select picker for built-in and MCP tools.
  • Advanced: Temperature, max tokens, timeout, retries, and API key reference.

Running Workflows

Click Run in the top toolbar or press Shift+Enter to execute the current workflow. You'll be prompted for input text (the InputNode payload), then the orchestrator begins execution.

  1. Input dialog: Type the initial text for your workflow and click Submit.
  2. Live execution: Each node card animates while processing. A green badge appears on completion.
  3. Streaming output: LLM responses stream in real-time. Tool calls and results appear inline.
  4. Completion: The Run Output panel shows the final result with cost breakdown.

Run History

Every workflow execution is saved. Open the Run History panel to browse past runs, view per-node traces, and replay any run from a specific step.

  • Replay: Re-run a past execution from any node in the chain.
  • Compare: View side-by-side outputs from different runs.
  • Export: Download run data as JSON for analysis.

Memory Panel

The Memory panel lets you ingest documents and search them semantically using the built-in RAG pipeline powered by sqlite-vec.

  • Ingest: Upload TXT, MD, PDF, or DOCX files. They're chunked, embedded, and stored in SQLite.
  • Search: Run semantic queries against your ingested documents.
  • Collections: Organize documents into named collections for targeted retrieval.
Note: See Memory & RAG in Core Concepts for the technical details on how embeddings and retrieval work.

Settings

The Settings panel (gear icon in the toolbar) gives you access to workspace configuration, provider management, and application preferences.

  • Providers: Add, test, and remove LLM providers. API keys are stored in the OS keychain.
  • Workspaces: Create and switch between workspaces. Each has its own SQLite database.
  • Remote Sidecar: Connect to a sidecar running on a different machine (with auth token).
  • Cost Alerts: Set a USD threshold to get warned when a single run exceeds it.
  • MCP Servers: Configure Model Context Protocol server connections.
