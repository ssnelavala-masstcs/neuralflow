Getting Started

This guide walks you through installing NeuralFlow, starting the development server, and running your first multi-agent workflow end-to-end.

Prerequisites

Install the following tools before you clone the repository. NeuralFlow is a Tauri v2 desktop application, so it requires a Rust toolchain and a JavaScript runtime in addition to the Python sidecar dependencies.

Requirement Minimum Version Notes
Node.js 20 LTS Use nvm or the official installer
Python 3.11+ 3.12 recommended; 3.10 and below are not supported
Rust toolchain 1.77+ Install via rustup — includes cargo
RAM 8 GB 16 GB recommended when running local models via Ollama
Operating system — macOS 12+, Ubuntu 22.04+, Windows 11 (with WebView2)
Note: On Windows, ensure the Microsoft Edge WebView2 runtime is installed. It ships with Windows 11 by default. Download it from the Microsoft website for older Windows 10 installs.

Install Rust

If you do not have Rust installed, run the official installer script.

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source "$HOME/.cargo/env"
rustup update stable

Install

Clone the repository, then install both the Python sidecar and the desktop frontend.

Clone the repository

git clone https://github.com/ssnelavala-masstcs/neuralflow.git
cd neuralflow

Install uv

NeuralFlow uses uv to manage the sidecar's Python environment. Install it globally with pip. 

pip install uv

Install sidecar dependencies

cd packages/sidecar
uv sync
cd ../..
Tip: uv sync reads pyproject.toml and creates a local .venv. You do not need to activate it manually — the uv run prefix activates it automatically at runtime.

Install pnpm

npm install -g pnpm

Install frontend dependencies

cd apps/desktop
pnpm install
cd ../..
Note: The first pnpm install also downloads the Tauri CLI and triggers a Rust compilation step. Expect 2–5 minutes on a modern machine.

Run in Development

Use the convenience script to start the sidecar and the Tauri desktop app together in a single terminal. The script streams both processes' logs side by side.

All-in-one script (recommended)

./scripts/dev.sh

The script waits for the sidecar to respond on http://127.0.0.1:7411/health before launching the Tauri window, so the canvas is never shown before the API is ready.

Manual: two terminals

Run each service independently when you need to attach a debugger or tail separate logs.

Terminal 1 — Python sidecar (port 7411)

cd packages/sidecar
uv run uvicorn neuralflow.main:app --host 127.0.0.1 --port 7411 --reload

Terminal 2 — Tauri desktop app (Vite on port 1420)

cd apps/desktop
pnpm tauri dev
Warning: Start the sidecar before Tauri. If the desktop app loads before port 7411 is listening, the canvas displays a connection error banner. Refresh the webview with Ctrl+R / Cmd+R once the sidecar is up.

Configure API keys

Provider API keys are read from environment variables at sidecar startup. Set them in your shell profile or in a .env file at the repository root. 

# .env — never commit this file
NEURALFLOW_KEY_OPENAI=sk-...
NEURALFLOW_KEY_ANTHROPIC=sk-ant-...
NEURALFLOW_KEY_GROQ=gsk_...

Your First Workflow

Follow these steps to create and run a minimal two-agent pipeline inside NeuralFlow.

  1. Open the application. After pnpm tauri dev finishes compiling, the NeuralFlow desktop window opens automatically and shows an empty canvas with a dot-grid background.
  2. Drag an Agent node. In the node palette on the left, find Agent under the Agent Nodes category. Drag it onto the canvas and drop it anywhere in the center.
  3. Configure the node. Click the node to select it. The Properties panel slides in from the right. Set the model field to gpt-4o and enter a system prompt, for example: "You are a research assistant. Summarize the input concisely."
  4. Drag a second Agent node. Drop another Agent node to the right of the first, leaving roughly 200 px of space between them.
  5. Connect the nodes. Hover over the right edge of the first node until a blue handle appears. Click and drag from that handle to the left edge of the second node. A directed edge with an animated arrow confirms the connection.
  6. Run the workflow. Click Run in the top toolbar (or press Shift+Enter). Type any text into the input dialog and click Submit. Each node animates while processing and shows a green badge when complete. The output appears in the Run Output panel.
Tip: Save your workflow with Ctrl+S / Cmd+S. NeuralFlow stores it in SQLite at ~/.neuralflow/workspaces/default/data.db. All run history is also persisted there and can be replayed from the Run History panel.

Troubleshooting

The table below covers the most common issues encountered during initial setup.

Issue Solution
Port 7411 already in use lsof -ti:7411 | xargs kill -9
Windows: netstat -ano | findstr :7411, then taskkill /PID <pid> /F
Keychain error on API key retrieval Set the NEURALFLOW_KEY_<PROVIDER> environment variable as a fallback. Example: export NEURALFLOW_KEY_OPENAI=sk-...
Blank canvas / white screen Check that Vite started on port 1420. On Linux, ensure webkit2gtk-4.1 is installed: sudo apt install webkit2gtk-4.1
uv sync fails with SSL error Set SSL_CERT_FILE to your CA bundle, or run uv sync --native-tls.
Rust compile error: "linker not found" Ubuntu/Debian: sudo apt install build-essential
macOS: xcode-select --install
Node drag from palette does nothing The drop event uses the dataTransfer key "application/neuralflow-node". Under Wayland, try DISPLAY=:0 ./scripts/dev.sh to force XWayland.
Still stuck? Open an issue at github.com/ssnelavala-masstcs/neuralflow/issues and include the output of ./scripts/dev.sh plus your OS and Node/Python/Rust versions.
