Node Reference

NeuralFlow has 10 node types arranged in 3 categories: Agent Nodes drive reasoning and control flow, Tool Nodes call external capabilities, and Data Nodes handle the flow of data through the graph.

Drag to add nodes. All node types live in the palette on the left side of the canvas. Drag any node onto the canvas to add it. Connect nodes by dragging from an output handle (right edge) to an input handle (left edge) of another node.

Agent Nodes

Agent nodes contain reasoning logic. They call LLMs, evaluate expressions, and pause execution for human review. Every agent node exposes an input handle on its left edge and one or more output handles on its right edge.

AgentNode

🤖

AgentNode LLM

Runs a single LLM inference call with a configurable system prompt, model, and tool list. Supports all providers available through LiteLLM, including OpenAI, Anthropic, Groq, Mistral, Gemini, DeepSeek, AWS Bedrock, Ollama, and any OpenAI-compatible endpoint.

PropertyTypeDescription
labelstringDisplay name shown on the canvas node.
modelstringLiteLLM model string, e.g. gpt-4o, claude-3-5-sonnet-20241022, ollama/llama3.
system_promptstringSystem message prepended to every inference call. Supports {"{{"}variable{"}}"} interpolation from upstream node output.
temperaturefloatSampling temperature (0.0–2.0). Default: 0.7.
max_tokensintegerMaximum completion tokens. Default: 2048. Set to 0 to use the model's context limit.
toolsstring[]List of built-in or MCP tool names to expose to this agent. The agent may call them during a single turn via tool-use / function-calling.
api_key_refstringKeychain key reference for the provider API key. If blank, falls back to the NEURALFLOW_KEY_<PROVIDER> environment variable.
timeout_sintegerPer-call timeout in seconds. Default: 120.
retriesintegerNumber of automatic retries on transient errors. Default: 2.

RouterNode

🔀

RouterNode Control

Evaluates a JavaScript expression against the incoming data object and routes execution to one of multiple named output handles. Use it to implement if/else branching, multi-way classification routing, and retry loops.

PropertyTypeDescription
labelstringDisplay name shown on the canvas node.
expressionstringJavaScript expression evaluated with data in scope. Must return a string matching one of the route names. Example: data.sentiment === 'positive' ? 'approve' : 'reject'
routesstring[]List of named output handles. Each name appears as a labelled port on the right edge of the node.
default_routestringRoute taken when the expression returns an unrecognised value or throws. Must be one of the names in routes.
Warning: The expression runs in a sandboxed V8 context with a 100 ms timeout. Avoid async operations or imports inside the expression string.

HumanNode

👤

HumanNode Pause

Pauses workflow execution and emits an awaiting_input SSE event. The frontend presents a modal to the operator, who types a response and clicks Submit. Execution resumes automatically with the submitted text injected into the next node's input. See Human-in-the-Loop for the full flow.

PropertyTypeDescription
labelstringDisplay name and heading shown in the modal.
promptstringInstruction or question displayed to the operator in the pause modal.
timeout_sintegerSeconds to wait before auto-cancelling the run. 0 means wait forever. Default: 0.
input_typeenumtext (plain text area) or approval (Approve / Reject buttons). Default: text.

Tool Nodes

Tool nodes call external capabilities — built-in utilities or remote MCP servers — and return structured results to the next node in the graph.

ToolNode

🔧

ToolNode Tool

Calls a single built-in or MCP tool explicitly, without going through an LLM agent. Use this when you want deterministic tool invocation rather than letting an agent decide. Built-in tools include web_search, file_read, file_write, http_request, code_exec, and shell.

PropertyTypeDescription
labelstringDisplay name shown on the canvas node.
tool_namestringName of the tool to call. Must exist in the tool registry. Built-in names are listed in packages/sidecar/neuralflow/tools/.
argumentsobjectStatic argument map merged with the runtime input. Static values take precedence over upstream data when keys conflict.
output_keystringKey under which the tool result is stored in the output object. Default: result.

McpNode

🔌

McpNode MCP

Connects to an external MCP (Model Context Protocol) server and calls one of its exposed tools. Supports both stdio and sse transports. The MCP connection pool is managed by the sidecar; connections are reused across nodes in the same run.

PropertyTypeDescription
labelstringDisplay name shown on the canvas node.
transportenumstdio or sse. For stdio, the sidecar spawns the MCP server as a subprocess. For sse, it connects to a running HTTP endpoint.
commandstring(stdio only) Shell command to launch the MCP server. Example: npx -y @modelcontextprotocol/server-filesystem /tmp
urlstring(sse only) Full URL of the MCP SSE endpoint. Example: http://localhost:3000/sse
tool_namestringName of the MCP tool to invoke. Must match a tool advertised in the server's capabilities response.
argumentsobjectStatic arguments merged with runtime input data before calling the tool.

Data Nodes

Data nodes shape, filter, and combine the data flowing through the graph. They contain no LLM calls and execute synchronously with negligible latency.

InputNode

📥

InputNode Entry

The workflow entry point. Exactly one InputNode is required per workflow. It receives the initial payload from the POST /api/runs request body and passes it downstream. A workflow with no InputNode cannot be run.

PropertyTypeDescription
labelstringDisplay name. Conventionally set to Input or the name of the expected payload field.
schemaobjectOptional JSON Schema object describing the expected input shape. Used for validation and the Properties panel's field hints. Not enforced at runtime.

OutputNode

📤

OutputNode Exit

Collects the final output of the workflow. Data arriving at an OutputNode is stored in the run record and streamed as a run_complete SSE event. A workflow may have multiple OutputNodes; all are collected into the run's outputs array.

PropertyTypeDescription
labelstringDisplay name. Appears as the key in the outputs map on the run record.
formatenumraw (pass data through unchanged), markdown (render in the output panel as Markdown), or json (pretty-print JSON). Default: raw.

TransformNode

⚙️

TransformNode Data

Applies a synchronous JavaScript transform to the incoming data object and passes the result downstream. Use it to reshape payloads, extract fields, format strings, or compute derived values without an LLM call.

PropertyTypeDescription
labelstringDisplay name shown on the canvas node.
codestringJavaScript function body. Receives data and must return the transformed value. Example: return { summary: data.output.trim().slice(0, 500) };
Note: The transform runs in a Node.js vm.runInNewContext sandbox with a 200 ms timeout. The return value replaces the entire data object passed to downstream nodes.

FilterNode

🔎

FilterNode Gate

Acts as a conditional gate. Evaluates a JavaScript predicate against the incoming data. If the predicate returns true, data passes through to the pass output handle. If it returns false, data is routed to the fail output handle (if connected) or the run step is marked as skipped.

PropertyTypeDescription
labelstringDisplay name shown on the canvas node.
predicatestringJavaScript expression that returns a boolean. data is in scope. Example: data.confidence > 0.8

AggregatorNode

🔗

AggregatorNode Fan-in

Waits for data to arrive on all connected input handles and combines them into a single object before passing execution downstream. Use it to join parallel branches — for example, after running several AgentNodes in parallel and merging their results.

PropertyTypeDescription
labelstringDisplay name shown on the canvas node.
strategyenum merge — shallow-merge all input objects into one.
array — collect all inputs into an array under the key items.
first — pass through the first input that arrives (race semantics). Default: merge.
wait_forintegerNumber of inputs to wait for before emitting output. 0 means wait for all connected inputs. Default: 0.
