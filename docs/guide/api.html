API Reference

The NeuralFlow sidecar exposes a REST API on http://127.0.0.1:7411. The interactive Swagger UI is available at localhost:7411/docs when the sidecar is running.

Overview

All endpoints are prefixed with /api. The API uses JSON for request and response bodies. Streaming endpoints use Server-Sent Events (SSE).

CategoryBase path
Workspaces/api/workspaces
Workflows/api/workflows

Authentication

By default, the sidecar accepts requests from 127.0.0.1 without authentication. If you enable remote access in Settings → Remote Sidecar, all requests must include a bearer token:

Authorization: Bearer <your-token>

Tokens are SHA-256 hashed before storage. Generate one in the settings UI or via the CLI.

Workspaces

GET /api/workspaces List all workspaces 
# Response
[
  {
    "id": "ws_default",
    "name": "default",
    "db_path": "~/.neuralflow/workspaces/default/data.db",
    "created_at": "2026-01-01T00:00:00Z"
  }
]
POST /api/workspaces Create a new workspace 
# Request
{ "name": "my-project" }

# Response 201
{ "id": "ws_abc123", "name": "my-project", "db_path": "..." }

Workflows

GET /api/workflows?workspace_id=<id> List workflows in a workspace 
[
  {
    "id": "wf_abc123",
    "name": "Research Assistant",
    "execution_mode": "sequential",
    "updated_at": "2026-03-15T10:30:00Z"
  }
]
POST /api/workflows Create or save a workflow 
# Request body
{
  "workspace_id": "ws_default",
  "name": "My Workflow",
  "execution_mode": "sequential",
  "nodes": [...],
  "edges": [...]
}
GET /api/workflows/<workflow_id> Get a workflow by ID (full node/edge graph)
PUT /api/workflows/<workflow_id> Update an existing workflow
DELETE /api/workflows/<workflow_id> Delete a workflow and all its runs

Runs and streaming

POST /api/runs Start a new workflow run 
# Request
{
  "workflow_id": "wf_abc123",
  "input": "Summarize the latest AI research papers"
}

# Response 202
{ "run_id": "run_xyz789", "status": "started" }
SSE /api/runs/<run_id>/stream Stream real-time run events via Server-Sent Events

Connect with EventSource. Each event has a type field:

# Event types
data: {"type": "node_start",    "node_id": "agent_1"}
data: {"type": "llm_chunk",     "text": "The latest..."}
data: {"type": "tool_call",     "tool": "web_search", "input": "..."}
data: {"type": "tool_result",   "result": "..."}
data: {"type": "node_complete", "node_id": "agent_1", "cost_usd": 0.0023}
data: {"type": "run_complete",  "output": "...", "total_cost_usd": 0.0047}
data: {"type": "run_error",    "error": "..."}
// JavaScript example
const es = new EventSource(`http://127.0.0.1:7411/api/runs/${runId}/stream`);
es.onmessage = (e) => {
  const event = JSON.parse(e.data);
  if (event.type === 'run_complete') es.close();
};
GET /api/runs?workflow_id=<id> List past runs for a workflow
GET /api/runs/<run_id> Get full run details including per-node traces
POST /api/runs/<run_id>/cancel Cancel a running execution
POST /api/runs/<run_id>/replay Replay a past run from a specific step 
{ "from_node_id": "agent_2" }

Providers

GET /api/providers?workspace_id=<id> List configured providers
POST /api/providers Add a provider (key stored in OS keychain) 
{
  "workspace_id": "ws_default",
  "provider_type": "openai",
  "api_key": "sk-...",
  "base_url": null
}
POST /api/providers/<provider_id>/test Test a provider connection 
# Response
{ "ok": true, "models": ["gpt-4o", "gpt-4o-mini", ...] }
DELETE /api/providers/<provider_id> Remove a provider and delete its key from the keychain

Tools

GET /api/tools List all available built-in tools and installed plugin tools
POST /api/tools/<tool_name>/invoke Directly invoke a tool (for testing) 
# Request — invoke web_search
{ "input": { "query": "NeuralFlow AI workflow" } }

# Response
{ "output": "[search results as text]", "duration_ms": 320 }

Memory

POST /api/memory/store Store text in a memory collection 
{
  "collection": "research_findings",
  "text": "LLMs show emergent capabilities at scale...",
  "metadata": { "source": "arxiv:2304.01234" }
}
POST /api/memory/search Semantic search over a memory collection 
{
  "collection": "research_findings",
  "query": "emergent behavior in large models",
  "top_k": 5
}

# Response
{
  "chunks": [
    { "text": "...", "score": 0.94, "metadata": {...} }
  ]
}

Templates

GET /api/templates List built-in and community templates
POST /api/templates/<template_name>/import Import a template as a new workflow 
curl -X POST \
  "http://127.0.0.1:7411/api/templates/research-assistant.json/import?workspace_id=ws_default"

Plugins

GET /api/plugins List installed plugins with their node types and load status
POST /api/plugins/reload Hot-reload all plugins without restarting the sidecar

Cost

GET /api/cost/summary?workspace_id=<id>&days=30 Get cost summary for the past N days 
{
  "total_usd": 1.234,
  "by_model": { "gpt-4o": 0.98, "claude-opus-4-5": 0.254 },
  "by_day":   { "2026-04-01": 0.12, "2026-04-02": 0.34 },
  "by_workflow": { "Research Assistant": 0.87 }
}
GET /api/cost/export.csv?workspace_id=<id> Download cost data as CSV

Versions

GET /api/versions?workflow_id=<id> List all saved versions of a workflow
POST /api/versions Save a new version snapshot 
{ "workflow_id": "wf_abc123", "label": "Before refactor" }
GET /api/versions/diff?v1=<id>&v2=<id> Get a diff between two workflow versions

MCP Servers

GET /api/mcp/servers List configured MCP server connections and their status
POST /api/mcp/servers Add an MCP server connection 
{
  "name":    "filesystem",
  "type":    "stdio",
  "command": "npx @modelcontextprotocol/server-filesystem ~/docs"
}

# or for SSE / HTTP
{
  "name": "my-mcp",
  "type": "sse",
  "url":  "http://localhost:8080/sse"
}

Error responses

All errors return a consistent JSON structure:

{
  "error": "workflow_not_found",
  "message": "No workflow with id wf_abc123 in this workspace",
  "status": 404
}
StatusMeaning
400Bad request — validation error in request body
401Unauthorized — missing or invalid auth token (remote mode only)
404Not found — resource does not exist
409Conflict — e.g. a run is already in progress for this workflow
422Unprocessable entity — Pydantic validation failed
429Rate limit exceeded
500Internal server error — check sidecar logs
Tip: The interactive API docs at localhost:7411/docs let you try every endpoint directly in your browser while the sidecar is running.
